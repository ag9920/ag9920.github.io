Hexo-Next-GithubPage

发表于 2022-01-01 更新于 2022-03-02 分类于 hexo 阅读次数： 阅读次数：

最近开始在github page 上写博客，有些自己在生活和工作上的感触，以及技术上学到的东西希望记录在这里。

现阶段只是个起步，不希望搞太复杂。看了下常见的方案，感觉还是 Next 的风格比较喜欢。简单调研了下搭了这个博客。这里记录下操作过程。这里的环境是 MacOS，可以参考下。

Step 1: Hexo

https://hexo.io/zh-cn/

hexo 是一个基于 Node.js 的博客框架，使用之前先准备好 npm 环境。参考官网示例执行即可，最后运行完 hexo server 后，在 localhost:4000 端口可以预览出博客效果。

详细的文档：https://hexo.io/zh-cn/docs/

Step 2: Next

先看效果：https://theme-next.js.org/plugins/

Next 是 Hexo 提供的一种主题，仓库地址：https://github.com/next-theme/hexo-theme-next

参照 https://theme-next.iissnan.com/getting-started.html#third-party-services 这里的步骤操作即可。注意 Next 使用 git clone 的时候仓库地址要使用上面这个最新的。文档里的仓库地址比较旧，记得替换。

文档里的「站点配置文件」就是指 _config.yml，有的是项目根目录的配置，有的是 Next theme 下的 _config.yml，注意区分。Muse，Mist，Pisces 三种 schema 都不错，根据个人喜好选择即可。

附上 next 的 FAQ：https://theme-next.iissnan.com/faqs.html

Step 3: Github Pages

上面两步完成后，经过简单的 _config.yml 属性调整（author，title，emal 等），你应该可以通过 hexo server 在本地启动你的博客了，博客内容放在 source/__posts 文件夹，默认会放一个 hexo 介绍的 Hello World。下面我们来和 github pages 进行集成。这一步参照 https://hexo.io/docs/github-pages.html 即可，是否使用 gh-pages 没有强要求。

重点关注One-command deployment 这一节，更新 _config.yml 中的 deploy 配置，改为你的 github 仓库地址。

执行 hexo clean && hexo deploy 命令后，hexo 会帮你处理部署的细节，把所有改动提交到远程仓库里。注意保证github 仓库的 pages 配置里的 branch 和 hexo 这里配置的 branch 一致即可。

首次执行时，github 会要求鉴权，你需要输入自己的账户和密码。由于github已经停止对密码登录的支持，此处需要输入 personal access token，可以通过 github developer settings 进行配置（https://github.com/settings/tokens）。参照 https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token

完成了这些配置后，日常情况下当你想要写一篇博客时，只需要进入本地仓库，执行 hexo new "My New Post" ，hexo会自动在 _posts 文件夹下创建一篇新的博客，包含了 title, date 和 tag（这些可以配置）。
